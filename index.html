1 Open the file & find where to edit

Everything you need to edit is in one place near the top of the script.

  • 1 Open novelore.html in any text editor (Notepad, VS Code, etc.)
  • 2 Press Ctrl + F (or Cmd + F on Mac) and search for: ✏️ YOUR CONTENT
  • 3 Everything between that comment and the STORAGE & THEME comment is where you do all your editing. Don't touch anything below that line.
💡 Tip: Save a backup of your file before editing — just duplicate it and name it novelore-backup.html.

2 Add your novel's details

Scroll down in the file until you find const novelDetails = {. This is the main database for every novel. Add a new entry inside the curly braces.

📖 novelDetails — template for one novel
// Inside novelDetails = { ... }, add this block.
// Replace everything in quotes with your own content.

'my-novel-id': {
  id:            'my-novel-id',   // must match the key above
  title:         'My Novel Title',
  author:        'Your Name',
  bg:            'linear-gradient(160deg,#C44B1A 0%,#3B1A08 100%)',
  emoji:         '⚔️',
  genres:        ['Dark Fantasy', 'Action'],
  status:        'Ongoing',          // 'Ongoing' or 'Completed'
  rating:        4.9,
  datePublished: 'January 1, 2026',
  synopsis: `Your synopsis goes here.
You can write multiple paragraphs.
Use backticks (the key above Tab) to wrap it.`,
  volumes: [
    // volumes go here — see Step 3
  ],
},
FieldWhat it does
idA short unique word with no spaces — used to link everything together. Example: 'shadow-throne'Required
titleThe full display title of your novel.Required
authorYour name or pen name.Required
bgThe gradient color used as the cover background. See Step 7 for color options.Required
emojiOne emoji that represents the novel. Shown on cards and lists.Required
genresArray of genre tags shown on the novel page. Can have 1–4 items.Required
statusEither 'Ongoing' or 'Completed'.Required
ratingA number from 1.0 to 5.0. You set this manually.Optional
datePublishedWhen you first published, shown as a tag. Any text works.Optional
synopsisThe book description. Use backtick quotes so you can write multiple lines freely.Required
coverImgURL to a cover image (https://...). If left out, the first letter of the title is shown instead.Optional
⚠️ Important: The id field and the key (the word before the colon at the top) must be exactly the same. If they don't match, clicking the novel won't work.

3 Add volumes and chapters

Inside the volumes: [ ] array of your novel, add one object per volume. Each volume contains an array of chapters.

📚 Volume structure
volumes: [
  {
    number: 1,
    title:  'The Tomb Unsealed',
    chapters: [
      // chapters go here — see below
    ],
  },
  {
    number: 2,
    title:  'Blood & Embers',
    chapters: [
      // more chapters
    ],
  },
],
📄 Chapter structure (one chapter)
{
  number:   1,                    // chapter number — must be unique across ALL volumes
  title:    'The Forgotten Name',
  date:     'Jan 1, 2026',
  volTitle: 'The Tomb Unsealed', // copy the volume title here too
  content: `Your chapter text goes here.

Write as many paragraphs as you want.
Leave a blank line between each paragraph.

Use *asterisks* around words for italics.

Use --- on its own line for a scene break.`,
},
💡 Chapter numbering: Chapter numbers must count up across the entire novel, not restart each volume. If Volume 1 has 5 chapters (1–5), Volume 2 starts at chapter 6. This is how the reader tracks your progress.
💡 Writing chapters: Wrap your content in backticks (`like this`). Inside, you can freely hit Enter for new lines, leave blank lines between paragraphs, and use *word* for italic text.
⚠️ Watch your quotes: If your chapter text contains a backtick character, you'll need to escape it with a backslash: \`. This is rare but worth knowing.

5 Add chapter updates & announcements

Every time you publish a new chapter, add it to updatesData. Add it at the top so the newest appears first.

🔄 Chapter update entry
const updatesData = [
  // newest chapter first ↑
  {
    novelId:      'my-novel-id',
    novelTitle:   'My Novel Title',
    emoji:        '⚔️',
    chapter:      13,           // must match the chapter number in novelDetails
    chapterTitle: 'Vol. 2 Ch. 8 — The Burning Gate',
    time:         'Just now',
  },
  // older entries below ↓
];
💡 Clicking an update takes the reader directly to that specific chapter. The chapter number here must match the number inside your chapter object in novelDetails.

For big news, add to announcementsData. Clicking an announcement opens the novel page.

📢 Announcement entry
const announcementsData = [
  // newest first ↑
  {
    title:   '⚔️ Vol. 3 Has Begun!',
    body:    'Volume 3 kicks off with a bang. New chapters weekly.',
    type:    'new',               // 'new', 'event', or 'system'
    date:    'May 1, 2026',
    novelId: 'my-novel-id',      // links the announcement to a novel
  },
];
type valueWhat badge it shows
'new'✨ New — orange badge
'event'🎉 Event — amber badge
'system'⚙️ System — grey badge

6 Add it to the hero banner (featured)

The hero slider at the top of the home page. You can have up to 5 slides. Find const heroSlides = [.

🌟 Hero slide entry
const heroSlides = [
  {

💡 Tag ideas: '🔥 Featured' · '⭐ Top Rated' · '🆕 Just Released' · '📖 Vol. 2 Live'

🔑 ID naming rules

  • ✓ Use only lowercase letters, numbers, and hyphens. Example: 'shadow-throne'
  • ✓ No spaces, no apostrophes, no special characters.
  • ✓ Keep it short and descriptive — it's only used in the code, readers never see it.
  • ✓ Must be unique — no two novels can share the same ID.

🚀 Checklist — adding a new novel

  • 1 Add the novel to novelDetails with id, title, author, bg, genres, synopsis, and volumes.
  • 2 Add the novel to at least one carousel array (trendingData, frenzyyyData, etc.).
  • 3 Add it to novelsNew if it's a fresh release (isNew: true).
  • 4 Add its chapters to updatesData (newest first).
  • 5 Optionally add it to heroSlides to feature it at the top.
  • 6 Save the file and open it in your browser to test.
