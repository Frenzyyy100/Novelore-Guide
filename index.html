Section 01

How NovéLore Stores Novels

NovéLore is a single HTML file. There's no database, no server, no backend. Every novel lives directly inside the JavaScript at the bottom of novelore.html as a plain JavaScript array called NOVELS.

This keeps things simple — you just edit the file, save it, push to GitHub Pages, and it's live. No accounts, no CMS, no hosting costs.

🥕

Where to find NOVELS: Open novelore.html in any text editor (VS Code recommended). Press Ctrl+F and search for const NOVELS. That's where all novels are defined.

novelore.html — simplified structure 
// Inside the <script> tag near the bottom of the file:
const NOVELS = [
  {
    id: "ashes-forgotten-king",
    title: "Ashes of the Forgotten King",
    author: "Frenzyyy",
    // ... more fields ...
    volumes: [
      {
        number: 1,
        title: "The Tomb Unsealed",
        chapters: [
          { number: 1, title: "The Dusty Gate", date: "Jan 01, 2026",
            content: `Your chapter text goes here...` }
        ]
      }
    ]
  }
];
Section 02

Adding Your Novel — Step by Step

1
Open novelore.html in VS Code
Right-click the file → "Open with Code". Never use Notepad — it doesn't handle large files well and can corrupt line endings.
2
Find the NOVELS array
Press Ctrl+F, search for const NOVELS = [. The array starts with [ and ends with ];. Each novel is an object { } inside it, separated by commas.
3
Add your novel object
Copy the template below and paste it inside the NOVELS array. Add a comma after the previous novel's closing } before your new one.
4
Fill in your details
Replace every placeholder with your actual novel data — title, author, synopsis, and most importantly, your chapter content using backtick strings.
5
Save and test locally
Save the file and open it in Chrome or Firefox. If your novel doesn't appear, open DevTools (F12) → Console. Any red error will tell you exactly what's wrong (usually a missing comma or unclosed backtick).
6
Push to GitHub Pages
Commit and push. Your novel is live within 1–2 minutes at your GitHub Pages URL.
Section 03

Novel Fields Reference

Here is every field a novel object can have. Required fields must be present or your novel won't load correctly.

Full Novel Template — Copy This 
{
  id:        "my-novel-slug",         // unique, lowercase, hyphens only
  title:     "My Novel Title",
  author:    "Your Name",
  cover:     "https://...",            // image URL or "" for placeholder
  synopsis:  "A short 2–3 sentence summary of your story.",
  genre:     "Fantasy",
  tags:      ["Magic", "Adventure", "Mystery"],
  status:    "Ongoing",                // "Ongoing" | "Completed" | "Hiatus"
  section:   "frenzyyy",               // which tab it appears in
  trending:  true,                     // shows in Trending tab
  hotpick:   false,
  updated:   "Feb 15, 2026",           // shown on the novel card
  volumes: [ /* see Section 4 */ ]
},
FieldRequiredDescription
idRequiredUnique identifier. Use lowercase letters and hyphens only, no spaces. Must be unique across all novels. Example: ashes-forgotten-king
titleRequiredFull display title of your novel.
authorRequiredYour name or pen name.
synopsisRequired2–4 sentence summary shown on the novel detail page. Keep it punchy and intriguing.
genreRequiredPrimary genre: Fantasy, Romance, Thriller, Sci-Fi, Horror, etc.
volumesRequiredArray of volume objects containing chapters. See Section 4.
coverOptionalURL to a cover image (HTTPS only). Leave as "" for a generated placeholder.
tagsOptionalArray of tag strings shown as chips. 3–6 tags recommended.
statusOptional"Ongoing", "Completed", or "Hiatus". Defaults to Ongoing if omitted.
sectionOptionalWhich tab section to appear in: "frenzyyy", "trending", "hotpick", or "dojo".
trendingOptionalSet true to include in the Trending tab.
hotpickOptionalSet true to include in Hot Picks.
updatedOptionalLast update date shown on the card. Example: "Feb 15, 2026".
Section 04

Volumes & Chapters Structure

Every novel has a volumes array. Each volume has a chapters array. You can have as many volumes and chapters as you want.

Volume + Chapter Structure 
volumes: [
  {
    number: 1,
    title:  "Volume Subtitle Here",
    cover:  "",              // optional volume-specific cover
    chapters: [
      {
        number:  1,
        title:   "Chapter Title Here",
        date:    "Jan 01, 2026",
        content: `
Your chapter text goes here.

Use blank lines to separate paragraphs.

You can write as much as you want inside
these backtick strings. They support
multiple lines natively.
        `
      },
      {
        number:  2,
        title:   "Second Chapter",
        date:    "Jan 08, 2026",
        content: `Chapter 2 content here...`
      }
    ]
  },
  {
    number: 2,
    title:  "Volume 2 Subtitle",
    chapters: [ /* more chapters */ ]
  }
]
⚠️

Chapter numbers must be globally unique across all volumes of a novel. Don't restart at 1 for each volume — keep counting up (Vol 1: chapters 1–8, Vol 2: chapters 9–16). The reader uses these numbers to track progress.

💡

Backtick strings (the ` character) are used for chapter content because they support line breaks. Regular quotes " do NOT — your text would all run together on one line.

Section 05

Writing & Formatting Chapter Content

Chapter content is plain text inside backtick strings. NovéLore automatically formats paragraphs — just leave a blank line between them, exactly like writing in a normal document.

¶ Paragraphs Automatic

Leave a blank line between paragraphs. The reader renders each paragraph as a separate block with proper spacing. No HTML needed.

— Scene Breaks Use ✦ or ***

To indicate a scene change, put ✦ or * * * on its own line between paragraphs. It renders as a visual divider for the reader.

❝ Dialogue Plain quotes

Use regular quotation marks for dialogue. "Hello," she said. Each character's line should be on its own paragraph for readability.

🧠 Inner Thoughts Italics tip

Wrap inner thoughts or emphasis in *asterisks* if the reader renders markdown, or just write them naturally in italics by pasting them directly.

⚠ Avoid backticks inside content Important!

Don't use the backtick character ` inside your chapter text — it closes the content string early and breaks the file. Use apostrophes or quotes instead.

Example Well-Formatted Chapter 
content: `The gates of the old city hadn't been touched in three
hundred years — or so they said.

Kael ran his fingers along the cold iron, feeling the rust
flake away like dried skin. Somewhere behind him, footsteps.

"You shouldn't be here," a voice said.

He didn't turn around.

✦

The morning had started like any other. Tea, bread, the
distant sound of the market carts rolling over cobblestone.
Nothing to suggest the world was about to split open.
`
Section 06

Cover Images

Covers make a huge visual difference. Here are your best options:

A
Use an image from the web
Right-click any image → "Copy image address". Paste the full HTTPS URL into the cover field. Make sure it's a tall/portrait image (2:3 ratio works best).
B
Upload to your GitHub repo
Create an /images/ folder in your repo. Upload your cover there. Then use a relative path: cover: "images/my-cover.jpg". This is the most reliable method.
C
Leave it blank for a placeholder
Set cover: "" and NovéLore will auto-generate a styled placeholder with your novel's initials and a gradient. No cover image needed.
⚠️

Avoid hotlinking covers from random sites — images can disappear or be blocked. GitHub, Imgur (direct link ending in .jpg/.png), or your own repo are the safest options.

Section 08

Publishing Checklist

Before you push to GitHub, run through this checklist:

  • Unique ID: My novel's id doesn't match any existing novel
  • Backtick strings: All chapter content fields use backticks `, not quotes
  • No backticks in text: My chapter text doesn't contain the ` character
  • Global chapter numbers: Chapter numbers don't restart at 1 in Vol 2+
  • Commas between novels: Each novel object in NOVELS is separated by a comma
  • Commas between chapters: Each chapter object is separated by a comma
  • Tested locally: Opened in browser, novel appears, chapter opens and reads correctly
  • No console errors: F12 → Console shows no red errors
  • Cover image loads: Cover image actually shows (not broken placeholder)
  • Synopsis written: Synopsis is filled in and not placeholder text
🥕

Common mistake: Forgetting the comma after the previous novel's closing } before adding yours. VS Code will underline it in red if you do — look for the squiggly line!
