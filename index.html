Where to Edit — The Golden Rule

Almost everything you'll ever need to change lives in one place: the content block at the very top of the <script> tag, clearly labelled ✏️ YOUR CONTENT. It starts around line 1143 and looks like this:

// ✏️  YOUR CONTENT

const heroSlides     = [];   // Featured banner slides
const trendingData   = [];   // 🔥 Trending section
const hotPicksData   = [];   // ⭐ Hot Picks section
const dojoDuelsData  = [];   // ⚔️ Dojo Duels section
const frenzyyyData   = [];   // ✍️ Frenzyyy's Work section
const novelsNew      = [];   // ✨ What's New tab
const updatesData    = [];   // 🔄 Updates tab
const announcementsData = [];// 📢 Announcements tab
const notificationsData = [];// 🔔 Notifications
const bookmarksData  = [];   // 🔖 Default bookmarks

Each array holds objects — one object per item. The square brackets [] mean it's currently empty. To add content, you place objects inside those brackets, separated by commas.

💡 Basic pattern: const myArray = [ { ...item1... }, { ...item2... } ]; — each item is wrapped in { }, fields separated by commas, items separated by commas.
⚠️ Don't edit below the content block. Everything beneath is the engine that powers the app. Touching it can break things. Stick to the arrays above.

Adding a Book / Novel Card

Novel cards appear in the carousel rows on the Home tab. There are four carousels — put your book in whichever one fits:

  • trendingData — 🔥 Trending (popular/viral reads)
  • hotPicksData — ⭐ Hot Picks (editor recommendations)
  • dojoDuelsData — ⚔️ Dojo Duels (action/battle novels)
  • frenzyyyData — ✍️ Frenzyyy's Work (your own writings)

Novel Card Fields

FieldTypeRequired?What it does
titletextRequiredThe book's title. Shows on the card and is used for the initial-letter fallback.
emojiemojiRequiredAn emoji that represents the book vibe. Used on the Update card.
bgCSS gradientRequiredCard background color/gradient (shown when no cover image, or as overlay).
ratingnumberRequiredScore out of 5.0, shown with a ★ on the card. e.g. 4.8
genretextRequiredGenre label shown under the rating. e.g. 'Fantasy'
statustextRequiredCompletion status. e.g. 'Ongoing' or 'Completed'
coverImgURLOptionalLink to a cover image. If omitted, shows the title's first letter instead.

Example — Adding Your First Book

Find const frenzyyyData = []; and change it to:

const frenzyyyData = [
  {
    title:    'The Shadow Realm',
    emoji:    '⚔️',
    bg:       'linear-gradient(160deg,#E8622A,#3B1A08)',
    rating:   4.8,
    genre:    'Fantasy',
    status:   'Ongoing',
  },
];

To add a second book, just add another object after the comma:

const frenzyyyData = [
  {
    title: 'The Shadow Realm', emoji: '⚔️',
    bg: 'linear-gradient(160deg,#E8622A,#3B1A08)',
    rating: 4.8, genre: 'Fantasy', status: 'Ongoing',
  },
  {
    title: 'Steel & Silence', emoji: '🗡️',
    bg: 'linear-gradient(160deg,#2A4A8A,#0A1A3A)',
    rating: 4.5, genre: 'Action', status: 'Completed',
  },
];
🎨 Card background tips: The first book in any array gets a special badge (🔥 Hot for trending, ⭐ Pick for hot picks). Make that one your best-looking entry. For the gradient, try linear-gradient(160deg, #STARTCOLOR, #ENDCOLOR) — pick colors that match the book's mood.

The Hero Banner (Featured Slide)

The hero banner is the big animated slider at the top of the Home tab. You can have up to 5 slides. If heroSlides is empty, a "Coming Soon" placeholder shows automatically.

Hero Slide Fields

FieldRequired?What it does
tagRequiredSmall label pill at the top. e.g. '🔥 Featured'
titleRequiredBig heading text.
descriptionRequiredShort blurb under the title.
bgRequiredSlide background gradient.
readHrefRequiredURL for the "Read Now" button. Use '#' as placeholder.
coverImgOptionalBook cover shown on the right side of the slide.
 
const heroSlides = [
  {
    tag:         '🔥 Featured',
    title:       'The Shadow Realm',
    description: 'A warrior wakes in a world between life and death — and nothing is what it seems.',
    bg:          'linear-gradient(135deg,#3B1A08 0%,#5A2510 55%,#C44B1A 100%)',
    readHref:    '#',          // ← swap '#' for your actual chapter URL later
    coverImg:    'https://yoursite.com/cover.jpg',  // ← optional
  },
];
📌 Slides auto-rotate every 5 seconds and users can swipe between them on mobile. The progress bar at the bottom shows timing. No extra setup needed.

What's New Tab

The ✨ What's New tab shows recently added novels in both a carousel (top) and a list view (below). An entry only shows in this tab if it has isNew: true.

FieldRequired?Notes
titleRequiredNovel title
emojiRequiredEmoji for the book
bgRequiredCard background gradient
descriptionRequiredShort blurb (shown in list view)
genreRequiredGenre tag label
statusRequired'Ongoing' or 'Completed'
ratingRequiredNumber out of 5
isNewRequiredMust be true to appear in the tab
coverImgOptionalCover image URL
 
const novelsNew = [
  {
    title:       'The Shadow Realm',
    emoji:       '⚔️',
    bg:          'linear-gradient(160deg,#E8622A,#3B1A08)',
    description: 'A warrior lost between worlds must choose a side before both collapse.',
    genre:       'Fantasy',
    status:      'Ongoing',
    rating:      4.8,
    isNew:       true,
  },
];

Chapter Updates Tab

The 🔄 Updates tab lists recently posted chapters. The first 3 entries automatically get a "NEW" badge.

FieldRequired?Notes
novelTitleRequiredName of the novel this chapter belongs to
emojiRequiredEmoji shown in the update circle icon
chapterRequiredChapter number (e.g. 12)
chapterTitleRequiredChapter name (e.g. 'The First Betrayal')
timeRequiredRelative time string, e.g. '2h ago' or 'Yesterday'
 
const updatesData = [
  {
    novelTitle:    'The Shadow Realm',
    emoji:         '⚔️',
    chapter:       12,
    chapterTitle:  'The First Betrayal',
    time:          '2h ago',
  },
  {
    novelTitle:    'Steel & Silence',
    emoji:         '🗡️',
    chapter:       5,
    chapterTitle:  'Echoes in the Dark',
    time:          'Yesterday',
  },
];
💡 Put your newest chapter first in the array — the list shows top to bottom and the first 3 get the NEW badge.

Announcements Tab

The 📢 Announce tab shows platform-wide news. There are 3 types: new, event, and system. Each shows with a different colored badge.

FieldRequired?Notes
titleRequiredAnnouncement headline
bodyRequiredThe announcement text body
typeRequiredOne of: 'new' (orange), 'event' (amber), 'system' (grey)

const announcementsData = [

Notifications

Notifications appear in the bell (🔔) mini-popup and the full notification drawer. Unread ones show a dot badge on the nav button. Setting read: false makes a notification appear as unread.

FieldRequired?Notes
msgRequiredThe notification message text
timeRequiredDisplay time, e.g. 'Just now', '3h ago'
readRequiredfalse = shows dot badge; true = already read
 
const notificationsData = [
  {
    msg:  '📢 Welcome to NovéLore! Start exploring novels.',
    time: 'Just now',
    read: false,
  },
  {
    msg:  '⚔️ Chapter 12 of The Shadow Realm is now live!',
    time: '2h ago',
    read: false,
  },
];

Using Book Cover Images

Every content type supports an optional coverImg field. If you leave it out, the app shows the novel's first capital letter as a stylized fallback — so you don't have to have a cover ready to launch.

Where to Host Your Cover Images

  • GitHub itself — Upload the image to your repo and use the raw URL: https://raw.githubusercontent.com/USERNAME/REPO/main/covers/mybook.jpg
  • ImgBB / Imgur — Free image hosting, just paste the direct image link
  • Any HTTPS URL — Must start with https:// (the app blocks non-secure links)

Cover Image Tips

  • Portrait orientation works best — roughly 2:3 ratio (e.g. 400×600px)
  • The app uses object-fit: cover, so the image fills the card without stretching
  • If the image fails to load (broken link, network error), the app automatically falls back to the letter initial — no blank boxes
  • Use loading="lazy" — images are already set to lazy-load, so page speed isn't hurt
{
  title:    'The Shadow Realm',
  emoji:    '⚔️',
  bg:       'linear-gradient(160deg,#E8622A,#3B1A08)',
  rating:   4.8,
  genre:    'Fantasy',
  status:   'Ongoing',
  coverImg: 'https://raw.githubusercontent.com/frenzyyy/novelore/main/covers/shadow-realm.jpg',
}

Quick Reference Cheat Sheet

When you're adding your first book, here's the step-by-step flow:

1
Open novelore.html in a text editor

VS Code is ideal. Ctrl+F (or Cmd+F) to search for ✏️ YOUR CONTENT to jump straight to the right spot.

2
Add your book to at least one carousel array

Start with frenzyyyData since it's your work. Copy the novel card template and fill in the fields.

3
Add a hero slide for your book Optional but cool

Add one entry to heroSlides. This makes your book the big featured banner when people open the app.

4
Add it to novelsNew with isNew: true

This makes it show up in the ✨ What's New tab so people see it as a fresh release.

5
Post a chapter update in updatesData

Every time you drop a new chapter, add an entry here. Put it first in the array for the NEW badge.

6
Announce it in announcementsData

Optional, but a quick type: 'new' announcement hypes up the launch.

7
Push to GitHub

Save the file, commit, and push. GitHub Pages will update within seconds.

Common Mistakes to Avoid

  • Missing comma between items — every object in an array needs a comma after its closing }, except the last one (though trailing commas are fine in JS)
  • Unclosed quotes — every string starts and ends with '. If your text contains an apostrophe (e.g. it's), use a backtick string instead: `it's fine`
  • Wrong type for type in announcements — only 'new', 'event', or 'system' are valid. Anything else falls back to 'new'
  • Using http:// for image URLs — the app only allows https://. Insecure links get blocked and show the letter fallback
  • Editing below the content block — if something suddenly breaks and you didn't touch the data arrays, you may have accidentally edited the app engine below
🔍 If the page goes blank or nothing shows, open your browser's DevTools (F12 → Console tab) and look for red error messages. The most common one is a syntax error — usually a missing comma or quote — and it'll tell you which line to fix.
