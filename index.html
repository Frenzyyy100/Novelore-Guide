Section 01

How NovéLore Stores Novels

NovéLore is a single HTML file. All novel data lives directly inside novelore.html as JavaScript objects — no database, no server, no backend. To add or update a novel, you edit the file and re-upload it.

There are two main places you'll edit when adding a novel:

A
novelDetails — the full novel data
This is a JavaScript object that holds everything about a novel: title, author, synopsis, cover image, all volumes, and all chapters. Every novel must have an entry here. Search for const novelDetails = { to find it.
B
Data arrays — which tabs it appears in
Separate arrays control which tabs and sections display your novel as a card: trendingData, hotPicksData, frenzyyyData, novelsNew, updatesData, and announcementsData. You add a short entry to whichever arrays apply to your novel.
📌

Think of novelDetails as the database record, and the data arrays as the display lists. Your novel needs to be in both places to show up and be readable.

Section 02

Adding Your Novel — Step by Step

1
Open novelore.html in VS Code
Right-click the file → "Open with Code". Never use Notepad — it doesn't handle large files well and can corrupt line endings.
2
Find novelDetails
Press Ctrl+F and search for const novelDetails = {. You'll see existing novels listed as 'novel-id': { ... }. Add yours after the last one, making sure there's a comma separating them.
3
Paste your novel template into novelDetails
Copy the full template from Section 3 and paste it inside the novelDetails object. Give it a unique key like 'my-novel'.
4
Add card entries to the right data arrays
Search for const frenzyyyData, const trendingData, const novelsNew, etc. Add a short card object for your novel to whichever arrays you want it to appear in. See Section 7 for details.
5
Fill in your chapter content
Replace every placeholder with your actual novel data — title, author, synopsis, and most importantly, your chapter content using backtick strings.
6
Save and test locally
Save the file and open it in Chrome or Firefox. If your novel doesn't appear, open DevTools (F12) → Console. Any red error will tell you exactly what's wrong — usually a missing comma or unclosed backtick.
7
Push to GitHub Pages
Commit and push. Your novel is live within 1–2 minutes at your GitHub Pages URL.
Section 03

novelDetails Fields Reference

Every novel lives inside the novelDetails object. Here is the full template and a breakdown of every field.

Full novelDetails Entry — Copy This 
'my-novel': {
  id:            'my-novel',           // must match the key above
  title:         'My Novel Title',
  author:        'Your Name',
  bg:            'linear-gradient(160deg,#C44B1A 0%,#3B1A08 100%)',
  emoji:         '📖',                 // shown as placeholder when no cover
  coverImg:      'https://i.imgur.com/yourimage.png', // or '' to use emoji
  genres:        ['Fantasy', 'Action', 'Adventure'],
  status:        'Ongoing',            // 'Ongoing' | 'Completed' | 'Hiatus'
  rating:        4.5,
  datePublished: 'January 01, 2026',
  synopsis:      `Your 2–4 sentence synopsis here.`,
  volumes: [ /* see Section 4 */ ]
},
FieldRequiredDescription
idRequiredUnique identifier. Must exactly match the object key. Use lowercase letters and hyphens only. Example: ashes-of-the-king
titleRequiredFull display title of your novel.
authorRequiredYour name or pen name.
bgRequiredCSS gradient string used as the card and novel page background. Use cssgradient.io to pick one.
emojiRequiredA single emoji shown as the fallback when no cover image is set.
synopsisRequired2–4 sentence summary shown on the novel detail page. Use a backtick string so it can span multiple lines.
genresRequiredArray of genre/tag strings shown as chips on the novel page. 2–4 recommended. Example: ['Dark Fantasy', 'Action']
volumesRequiredArray of volume objects containing chapters. See Section 4.
coverImgOptionalDirect URL to a cover image (must end in .jpg, .png, etc). Leave as '' to use the emoji placeholder instead.
statusOptional'Ongoing', 'Completed', or 'Hiatus'. Defaults to Ongoing if omitted.
ratingOptionalA number from 0.0 to 5.0 shown as a star rating on cards.
datePublishedOptionalPublication date shown on the novel detail page. Example: 'January 17, 2026'
Section 04

Volumes & Chapters Structure

Every novel has a volumes array inside its novelDetails entry. Each volume has a chapters array. You can have as many volumes and chapters as you want.

Volume + Chapter Structure 
volumes: [
  {
    number: 1,
    title:  'Volume Subtitle Here',
    chapters: [
      {
        number:  1,
        title:   'Chapter Title Here',
        date:    'Jan 01, 2026',
        content: `
Your chapter text goes here.

Use blank lines to separate paragraphs.

You can write as much as you want inside
these backtick strings.
        `
      },
      {
        number:  2,
        title:   'Second Chapter',
        date:    'Jan 08, 2026',
        content: `Chapter 2 content here...`
      }
    ]
  },
  {
    number: 2,
    title:  'Volume 2 Subtitle',
    chapters: [ /* more chapters */ ]
  }
]
⚠️

Chapter numbers must be globally unique across all volumes of a novel. Don't restart at 1 for each volume — keep counting up (Vol 1: chapters 1–8, Vol 2: chapters 9–16). The reader uses these numbers to track progress and the NEW tag.

💡

Backtick strings (the ` character) are used for chapter content because they support line breaks. Regular quotes " do NOT — your text would all run together on one line.

Section 05

Writing & Formatting Chapter Content

Chapter content is plain text inside backtick strings. NovéLore automatically formats paragraphs — just leave a blank line between them, exactly like writing in a normal document.

¶ Paragraphs Automatic

Leave a blank line between paragraphs. The reader renders each paragraph as a separate block with proper spacing. No HTML needed.

— Scene Breaks Use ---

To indicate a scene change, put --- on its own line between paragraphs. It renders as a centered "· · ·" visual divider for the reader.

❝ Dialogue Plain quotes

Use regular quotation marks for dialogue. "Hello," she said. Each character's line should be on its own paragraph for readability.

🧠 Inner Thoughts / Emphasis *asterisks*

Wrap text in *asterisks* to render it in italics. Example: *This is wrong,* she thought.

⚠ Avoid backticks inside content Important!

Don't use the backtick character ` inside your chapter text — it closes the content string early and breaks the file. Use apostrophes or quotes instead.

Example Well-Formatted Chapter 
content: `The gates of the old city hadn't been touched in three
hundred years — or so they said.

Kael ran his fingers along the cold iron, feeling the rust
flake away like dried skin. Somewhere behind him, footsteps.

"You shouldn't be here," a voice said.

He didn't turn around.

---

The morning had started like any other. Tea, bread, the
distant sound of the market carts rolling over cobblestone.
Nothing to suggest the world was about to split open.
`
Section 06

Cover Images

Covers make a huge visual difference. The field name is coverImg — make sure you use this exact name everywhere.

A
Upload to Imgur (recommended)
Go to imgur.com, upload your image, then right-click it → "Copy image address". The URL must end in .png or .jpg (e.g. https://i.imgur.com/abc123.png). Paste it as the coverImg value.
B
Upload to your GitHub repo
Create an /images/ folder in your repo. Upload your cover there. Then use a relative path: coverImg: 'images/my-cover.jpg'. This is the most reliable long-term method.
C
Leave it blank for an emoji placeholder
Set coverImg: '' and NovéLore will auto-generate a styled placeholder using your novel's emoji and bg gradient. No image needed.
⚠️

Add coverImg in every place the novel appears — inside novelDetails, and in each data array entry (trendingData, frenzyyyData, novelsNew, etc). If you miss one, the card will show the emoji placeholder instead of the cover.

Section 07

Adding to Tabs & Feeds

Beyond novelDetails, you add short card objects to separate arrays that control which tabs and feeds display your novel. Each array is clearly labeled with a comment in the file.

ArrayWhere it showsRequired fields
trendingData🔥 Trending tabtitle, novelId, emoji, bg, rating, genre, status, coverImg
hotPicksData⭐ Hot Picks tabtitle, novelId, emoji, bg, rating, genre, status, coverImg
frenzyyyData✍️ Frenzyyy's Work tabtitle, novelId, emoji, bg, rating, genre, status, coverImg
novelsNew✨ What's New tabtitle, novelId, emoji, bg, description, genre, status, rating, isNew, coverImg
updatesDataChapter updates feednovelId, novelTitle, emoji, chapter, chapterTitle, time
announcementsDataAnnouncements feedtitle, body, type, date, novelId
Example — Adding to frenzyyyData and novelsNew 
// Frenzyyy's Work tab card
const frenzyyyData = [
  { title:'My Novel', novelId:'my-novel', emoji:'📖',
    bg:'linear-gradient(160deg,#C44B1A 0%,#3B1A08 100%)',
    rating:4.5, genre:'Fantasy', status:'Ongoing',
    coverImg:'https://i.imgur.com/yourimage.png' },
];

// What's New tab card
const novelsNew = [
  { title:'My Novel', novelId:'my-novel', emoji:'📖',
    bg:'linear-gradient(160deg,#C44B1A 0%,#3B1A08 100%)',
    description:'A short tagline for the novel.',
    genre:'Fantasy', status:'Ongoing', rating:4.5,
    isNew:true, coverImg:'https://i.imgur.com/yourimage.png' },
];
📌

The announcementsData type field accepts three values: 'new' (orange), 'event' (amber), or 'system' (grey). Pick whichever fits the announcement.

Section 08

The NEW Tag & Chapter Updates

The NEW badge on the chapter list is automatic — it always appears on only the latest chapter (the one with the highest chapter number). You don't need to set anything. When you add a new chapter, the badge moves to it automatically.

To announce a new chapter in the Updates feed, add an entry to updatesData. Always add new entries at the top of the array so the most recent update appears first.

Adding a Chapter Update 
const updatesData = [
  // ← newest entries go at the TOP
  { novelId:'my-novel', novelTitle:'My Novel', emoji:'📖',
    chapter:3, chapterTitle:'Vol. 1 Ch. 3 — The Title',
    time:'Apr 06, 2026' },
  // older entries below...
];
🥕

The first 3 entries in updatesData automatically get a NEW pill badge on the updates feed. So always put your freshest chapter update at position 1.

Section 09

Publishing Checklist

Before you push to GitHub, run through this checklist:

  • Unique ID: My novel's id in novelDetails doesn't match any existing novel
  • Key matches ID: The novelDetails key (e.g. 'my-novel') exactly matches the id field inside
  • Backtick strings: All chapter content fields use backticks `, not quotes
  • No backticks in text: My chapter text doesn't contain the ` character
  • Global chapter numbers: Chapter numbers don't restart at 1 in Vol 2+
  • Commas between entries: Every object inside novelDetails and data arrays is separated by a comma
  • coverImg in all arrays: I've added coverImg to every array entry, not just novelDetails
  • updatesData updated: I've added the new chapter to updatesData at the top of the array
  • Tested locally: Opened in browser, novel appears, chapter opens and reads correctly, scrolls to top on Next/Prev
  • No console errors: F12 → Console shows no red errors
  • Cover image loads: Cover image actually shows (not broken placeholder)
  • Synopsis written: Synopsis is filled in and not placeholder text
🥕

Common mistake: Forgetting a comma after the previous novel's closing } before adding yours. VS Code will underline it in red — look for the squiggly line!
